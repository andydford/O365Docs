Installing NuGet package

The NuGet package is hosted in the DevDDA DevOps artifact library. In order to install the package from Visual Studio you will need to add the artifact repository to your list of NuGet sources.

Open the NuGet Package Manager -> Package Manager Settings from the Tools menu in Visual Studio and select Package Sources.

Click on the and enter the following in the Name and Source text boxes respectively:

Name = DevDDA
Source = https://pkgs.dev.azure.com/DevDDA/_packaging/DevDDA/nuget/v3/index.json

Once done click on Update then OK. This feed is now available in the NuGet package viewer as shown below.

In order for the NuGet package manager to be able to find DevDDA packages make sure you have selected DevDDA from the settings or All.

Select Browse in the package manager view and search for Hmrc.BDApp.O365. This should display the package but if not make sure which NuGet repository you have set Package source to.

Depending on your Visual Studio settings you may be presented with the following dialogue asking the desired format of the package manager.

Select the option shown, PackageReference in project file.

This package will not install if your project has not been converted to PackageReference.
