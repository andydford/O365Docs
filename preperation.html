Preperation

The Hmrc.Bdapp.O365 package is built using the ProjectReferences configuration and as such needs to be installed as a PackageReference and not Package.config

If your existing project has a packages.config then in the solution explorer right click on the packages.config of the project you wish to install the package and select Migrate packages.config to PackageReference...

This will display a dialogue showing existing packages and if there are any issues.

It is recommended that you refer to Microsoft Graph API on the Microsoft documentation site. Here you can read as to why specifying new future projects to use PackageReference is a best practice and also the benefits for source control etc. This does not apply to ASP.NET or C++ projects.

New projects should be created as PackageReference.
